## Problem Statement

Người dùng đậu JLPT N3 cách đây 7 năm, đã quên nhiều và muốn **học lại bài bản, có lộ trình** lên N2 → N1. Bối cảnh: dev IT có thể đang/sắp làm việc với khách Nhật.

**Vấn đề với các app sẵn có:**
- **Anki / WaniKani:** rất tốt cho ghi nhớ vocab/kanji nhưng KHÔNG có curriculum, không hướng dẫn "hôm nay học gì", không có giải thích bilingual
- **Bunpro:** có curriculum N2/N1 ổn nhưng tiếng Anh, không hỗ trợ JP-VN, không có AI helper
- **Duolingo:** quá nông, không tới N2
- **JapanesePod101:** lỗi thời, audio-first, không structured drill
- **Tất cả:** không có gì tích hợp riêng cho dev IT business JP

**Cốt lõi nhu cầu:**
1. Lộ trình rõ ràng theo tuần — "tuần này học gì, ngày này học gì"
2. Bilingual JP-VN và VN-JP — không phải tiếng Anh trung gian
3. Bài tập ôn luyện có feedback ngay
4. Nhắc nhở học đều — vì người đi làm dễ quên duy trì
5. AI helper trong context bài học — hỏi khi không hiểu, không phải tool độc lập
6. Nội dung phù hợp người **học lại** sau 7 năm gián đoạn (không phải zero, cũng không phải fluent)

App này là **công cụ cá nhân**, deploy public web app, password đơn bảo vệ, miễn phí 100%.

## What Makes This Cool

**App này khác Bunpro/WaniKani ở 3 điểm:**

1. **Bilingual JP↔VN native, không qua tiếng Anh** — mọi grammar/vocab có giải thích tiếng Việt thuần. Khi user gõ tiếng Việt hỏi "ngữ pháp này dùng khi nào?", AI trả lời tiếng Việt với ví dụ tiếng Nhật. Khi gõ tiếng Nhật, AI trả tiếng Nhật.

2. **Curriculum-driven nhưng linh hoạt** — Có lộ trình 12 tuần mặc định cho N2 (chia theo chủ đề: ngữ pháp nguyên nhân-kết quả, ngữ pháp giả định, kính ngữ business...) nhưng user có thể skip/redo bất cứ lúc nào. Mỗi ngày có "Bài hôm nay" tự động chọn dựa trên progress.

3. **AI helper trong context, không phải chat tự do** — AI sidebar hiển thị NGAY trong trang bài học hiện tại, biết chính xác user đang học mục nào, tự động lấy context. User không phải copy/paste hay giải thích lại.

**Tích hợp IT business sau N2:** Khi user đã master phần lớn N2 grammar, hệ thống mở khóa "IT track" — vocab/grammar/email mẫu đặc thù phát triển phần mềm.

**Hooked-on factor:** Mở app, thấy "Bài 5/60 — hôm nay 〜ものの, 〜ながらも + 8 vocab". 25 phút làm xong, có quiz feedback. Reminder ngày mai. Cảm giác **tiến bộ đo được**, không lan man.

## Constraints

- **100% free.** Hosting, AI API, DB đều free tier.
- **Single user.** Password bảo vệ đơn giản.
- **Bilingual JP↔VN bắt buộc.** Không phải optional.
- **Curriculum-driven là core**, không phải nice-to-have.
- **Mobile + laptop.** Chạy được trên iPhone Safari/Android Chrome (PWA).
- **MVP 2-3 weekend.** Honest budget: 2 weekend cho chỉ "daily lesson + bilingual + AI helper", 3 weekend nếu thêm quiz + reminder.
- **Trình độ user:** N3 cách 7 năm, có nền nhưng cần hệ thống lại.

## Approaches Considered

### Approach A (CHOSEN): Curriculum-first daily lesson app
- Trang chính: "Bài hôm nay" — 3-5 grammar + 8-10 vocab + ví dụ song ngữ + audio + mini quiz
- AI helper sidebar trong context bài học
- Curriculum 12 tuần N2 hand-curated (sau bồi N1)
- SRS cho items quiz sai
- Daily reminder qua PWA notification hoặc email
- **Effort:** ~3 weekend
- **Risk:** Med — curriculum design tốn thời gian; AI quality cho giải thích sắc thái có thể yếu

### Approach B: Chat-first với curriculum mềm
- Như version cũ: chat tutor là interface chính, có "topic of the day" gợi ý
- Curriculum mỏng, AI dẫn dắt
- **Effort:** ~2 weekend
- **Trade-off:** không match cái user nói cần ("lịch trình rõ ràng") — chat-first vẫn lan man

### Approach C: Pre-built lesson player (zero AI)
- Curriculum hand-curated chi tiết, chạy như giáo trình tĩnh
- Không AI, không LLM, chỉ HTML+JS+JSON
- **Effort:** ~4 weekend (vì content tốn thời gian)
- **Trade-off:** mất khả năng AI giải thích; curriculum sẽ cứng

## Recommended Approach (A — Curriculum-first daily lesson app)

### Tech stack
- **Frontend:** Next.js 15 + React 19 + Tailwind + shadcn/ui, PWA-enabled
- **Hosting:** Cloudflare Workers qua OpenNext adapter (`npm create cloudflare@latest -- --framework=next`)
- **AI Models on Workers AI:**
  - **Primary:** `@cf/meta/llama-3.3-70b-instruct-fp8-fast` — AI helper + quiz generator
  - **Fallback (eng-review OV4):** `@cf/meta/llama-3.1-8b-instruct-fast` — auto-fallback nếu primary deprecated/error. Lưu trong `lib/ai-models.ts` constants.
  - `@cf/openai/whisper-large-v3-turbo` — STT (post-MVP)
  - `@cf/myshell-ai/melotts` — TTS đọc câu mẫu (Day-0 Spike để confirm JP support)
- **TTS Fallback:** Browser `SpeechSynthesis` API với voice JP — default cho MVP an toàn
- **TTS Cache (eng-review P1):** Cloudflare Cache API built-in. Cache key = `tts:ja:${sha256(phrase)}`. Save 80% neurons cho replay shadowing.
- **Database:** Cloudflare D1 (SQLite-on-edge), curriculum stored trong D1 từ MVP (eng-review C1).
- **Reminder:** PWA Web Push (eng-review OV1 confirm). Detailed spec ở section "PWA Push Implementation" bên dưới.
- **Auth:** HTTP Basic Auth qua Next.js middleware. **Manual constant-time compare** (eng-review A4 — Web Crypto không có `timingSafeEqual` built-in trên edge):

  ```typescript
  function timingSafeEqual(a: string, b: string): boolean {
    if (a.length !== b.length) return false;
    let result = 0;
    for (let i = 0; i < a.length; i++) {
      result |= a.charCodeAt(i) ^ b.charCodeAt(i);
    }
    return result === 0;
  }
  ```
  Matcher: `['/((?!api/health|_next/static|manifest.json|sw.js).*)']`.

### Curriculum architecture (eng-review C1: D1-based từ MVP, eng-review A1: date-based scheduling)

**Cấu trúc:** Curriculum được lưu trong D1 bảng `lessons`, không bake bundle. Mỗi lesson có `scheduled_date` để hỗ trợ làm bù.

**Migration strategy:** File JSON nguồn (`curriculum/n2/week-XX.json`) trong repo, migration script `scripts/import-curriculum.ts` đọc files → INSERT vào D1 lần đầu deploy. Update lesson = re-run migration (idempotent UPSERT theo `id`).

**Today's lessons query:**
```sql
SELECT id, content_json FROM lessons
WHERE scheduled_date <= ? -- today VN time
  AND id NOT IN (
    SELECT lesson_id FROM user_progress
    WHERE user_id = 'me' AND status = 'completed'
  )
ORDER BY scheduled_date ASC
LIMIT 5;  -- catch-up cap khi nghỉ nhiều ngày
```

Hiển thị: lesson đầu tiên = "Hôm nay", còn lại = "Bài còn nợ" (UI accordion).

#### Accessibility Specs (design-review R2)

- **Touch targets:** All interactive elements ≥ 44px × 44px on mobile (TTS audio, quiz options, navigation).
- **Lang attributes:** Every JP text wrapped in `<span lang="ja">`, VN in `<span lang="vi">`. Screen reader đọc đúng accent.
- **Keyboard navigation:**
  - Quiz: arrow keys ↑↓ navigate options, Enter submit
  - Lesson: Tab flow logical (grammar 1 → audio → compare → grammar 2 → ...)
  - `V` keyboard shortcut: toggle bilingual VN visibility
  - `?` show keyboard shortcuts help modal
- **Focus ring:** asagi 2px outline, visible cho mọi focusable element. Không dùng default browser blue.
- **Color contrast (WCAG AA):**
  - Body text on paper: ink (#1a1a1a) on #fafaf7 = 16.4:1 ✓
  - Muted text: 8a8a85 on #fafaf7 = 4.51:1 ✓ (vừa pass)
  - Asagi on paper: 4.91:1 ✓
  - Yellow warning bar: dùng warning color #c8941d on dark text, không light text trên yellow
- **Dark mode:** Toggle theo system preference (`@media prefers-color-scheme`), override trong Settings (Auto/Light/Dark), persist localStorage. All colors có dark variant trong yaml trên.

### `callAI()` wrapper helper (eng-review C3)

File `lib/ai.ts` cung cấp wrapper duy nhất cho mọi AI call (chat, quiz gen, STT, TTS). Pattern:

```typescript
export async function callAI<T>(opts: {
  model?: string;  // default MODEL_PRIMARY
  prompt: string | Message[];
  stream?: boolean;
  validate?: (raw: unknown) => T;  // Zod parser cho structured output
}): Promise<{ data: T } | { error: AIError }> {
  const controller = new AbortController();
  const timeoutId = setTimeout(() => controller.abort(), 30_000);
  try {
    return await tryWithRetry(opts, controller.signal, {
      maxRetries: 1,
      retryOn: [500, 502, 503],
      modelFallback: MODEL_FALLBACK,  // tự switch nếu primary deprecated
    });
  } catch (err) {
    return { error: friendlyError(err) };
  } finally {
    clearTimeout(timeoutId);
  }
}
```

Friendly errors:
- 429 → "Hết quota hôm nay, mai thử lại 7h sáng (UTC reset 00:00)"
- timeout → "AI phản hồi chậm. Thử lại?"
- 5xx sau retry → "Server có vấn đề, thử lại sau vài phút"
- model deprecated → silent fallback sang MODEL_FALLBACK + log

### SRS algorithm — SM-2 (eng-review OV3)

Áp dụng cho item user trả lời sai trong quiz. Lưu trong `srs_queue` table.

```typescript
// lib/srs.ts
const MIN_EASE = 1.3;
const MAX_EASE = 2.5;
const GRADUATE_INTERVAL_DAYS = 30;
const LEECH_THRESHOLD = 5;

export function scheduleNextReview(item: SrsItem, correct: boolean): SrsItem {
  if (correct) {
    const newInterval = Math.round(item.interval_days * item.ease_factor);
    return {
      ...item,
      interval_days: newInterval,
      ease_factor: Math.min(MAX_EASE, item.ease_factor + 0.1),
      next_review_date: addDays(today(), newInterval),
      graduated: newInterval > GRADUATE_INTERVAL_DAYS,
    };
  } else {
    return {
      ...item,
      interval_days: 1,
      ease_factor: Math.max(MIN_EASE, item.ease_factor - 0.2),
      next_review_date: addDays(today(), 1),
      failures: item.failures + 1,
      is_leech: item.failures + 1 >= LEECH_THRESHOLD,
    };
  }
}

// Daily review query
export async function getReviewsDue(d1: D1Database, today: string) {
  return d1.prepare(
    `SELECT * FROM srs_queue
     WHERE next_review_date <= ?
       AND graduated = 0 AND is_leech = 0
     ORDER BY next_review_date ASC`
  ).bind(today).all();
}
```

- Daily lesson view shows "Hôm nay học bài + Ôn tập (N items)"
- "Ôn tập" → list items due, click vào hiển thị grammar/vocab + 1 quick quiz
- Trả lời → call `scheduleNextReview()` → update D1
- Leech items → tag UI badge "Cần học lại từ đầu", show riêng
